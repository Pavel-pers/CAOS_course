# C-strings

В этой задаче вам нужно реализовать несколько функций для работы с C-строками. Напомним, что
строками в C являются указатели на начало массива символов. Содержимым строки считаются символы
от начала массива до первого нулевого символа (`\0`).

Реализуйте аналоги библиотечных функций

- [`StrLen`](https://www.man7.org/linux/man-pages/man3/strlen.3.html) – вычисляет длину переданной строки
- [`StrNLen`](https://man7.org/linux/man-pages/man3/strnlen.3.html) – вычисляет длину переданной, учитывая не более `limit` байт
- [`StrCmp`](https://www.man7.org/linux/man-pages/man3/strcmp.3.html) – лексикографически сравнивает две строки. Возвращает отрицательное число, если первая строка меньше, ноль, если они равны, положительное число, если первая строка больше.
- [`StrNCmp`](https://man7.org/linux/man-pages/man3/strncmp.3p.html) – то же самое, но при проверке учитывается не более, чем `limit` первых символов каждой из строк
- [`StrCat`](https://man7.org/linux/man-pages/man3/strcat.3p.html) – конкатенирует `s1` и `s2`, дописывая содержимое `s2` в конец `s1`. Операция происходит in-place, новая память не выделяется
- [`StrStr`](https://man7.org/linux/man-pages/man3/strstr.3.html) – ищет первое вхождение строки `needle` в строку `haystack`, возвращает `nullptr`, если вхождений нет. Допускается реализовать алгоритм за $O(|needle| \cdot (|haystack| - |needle| + 1))$.
- [`StrDup`](https://www.man7.org/linux/man-pages/man3/strdup.3.html) – динамически выделяет память и создает копию строки `str`
- [`StrNDup`](https://www.man7.org/linux/man-pages/man3/strdup.3.html) – то же, что и `StrDup`, но копирует не более `limit` первых символов.
- `AStrCat` – возвращает результат конкатенации строк `s1` и `s2` в виде новой динамически аллоцированной строки.
- `Deallocate` – освобождает память, выделенную аллоцирующими методами (в зависимости от выбранного вами метода аллокации памяти).

Использовать функции для работы со строками из стандартной библиотеки запрещено.

Все функции, не совершающие динамических аллокаций, следует реализовать в файле `no-alloc.cpp`, они будут
собираться [без стандартной библиотеки](../../../docs/nostd.md).

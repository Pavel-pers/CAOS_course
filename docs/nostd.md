# Nostd

В задачах этого типа ваш код будет собираться без стандартной библиотеки. Это в том числе означает,
что большинство библиотечных функций использовать не получится. В частности, нельзя использовать:

- `std::cin/std::cout`
- `operator new/operator delete`
- `malloc/free`
- Обёртки для системных вызовов из `glibc` (`open/close/read/write/mmap/...`)
- Исключения
- Глобальные переменные с нетривиальным конструктором – глобальные переменные всегда инициализируются нулями

Большинство констант, определенных в заголовочных файлах `glibc`, использовать можно,
подключив соответствующий заголовочный файл (например `<sys/mman.h>`). Это не гарантируется,
но фактически работает с большинством компиляторов.

Для выполнения системных вызовов используйте методы из заголовочного файла
[`syscalls.hpp`](../common/syscalls/include/syscalls.hpp). Для дебажного вывода
используйте функцию `nostd::Print` из заголовочного файла [`io.hpp`](../common/nostd/include/io.hpp).
